# Scenario 01: Pure Navier-Stokes Flow (No Chemistry)

## Overview

This scenario solves **only the Navier-Stokes equations** for fluid flow through a parallel plate channel. No substrates are defined, no reactions occur, and no biomass is present. It serves as the simplest possible CompLaB3D validation: confirming that the LBM flow solver produces the correct analytical Poiseuille velocity profile.

## Reactions

None. This is a pure hydrodynamics test.

## Input Parameters

| Parameter | Value | Units | Description |
|-----------|-------|-------|-------------|
| `delta_P` | 2.0e-3 | (lattice) | Pressure drop across the domain |
| `tau` | 0.8 | (lattice) | LBM relaxation time |
| `dx` | 1 | um | Voxel size |
| `biotic_mode` | false | -- | No microbes |
| `enable_kinetics` | false | -- | No biotic kinetics |
| `enable_abiotic_kinetics` | false | -- | No abiotic kinetics |
| `number_of_substrates` | 0 | -- | No solutes defined |

## Domain

| Property | Value |
|----------|-------|
| Dimensions (nx x ny x nz) | 30 x 15 x 15 |
| Geometry type | Parallel plate channel |
| Description | Solid walls on the top and bottom (bounce-back), open sides (periodic or symmetry), flow driven in the x-direction by `delta_P` |
| Material numbers | 0 = solid, 1 = bounce-back walls, 2 = pore space |

## Boundary Conditions

| Boundary | Type | Value |
|----------|------|-------|
| Inlet (x = 0) | Pressure (higher) | Set by `delta_P` |
| Outlet (x = nx) | Pressure (lower) | Reference pressure |
| Top wall (z = nz) | Bounce-back (no-slip) | Solid wall |
| Bottom wall (z = 0) | Bounce-back (no-slip) | Solid wall |
| Sides (y = 0, y = ny) | Open / periodic | No lateral walls |

## Validation Criteria

The reviewer should check the following:

1. **Parabolic velocity profile.** Extract the velocity along a vertical line at the channel midpoint (x = 15, y = 7). The x-velocity component should form a parabola with maximum at the centerline (z = 7) and zero at both walls.

2. **Centerline maximum velocity.** The maximum velocity should occur at the geometric center of the gap between the plates. The magnitude depends on `delta_P`, `tau`, and the gap width; it should match the Poiseuille prediction:
   ```
   u_max = (delta_P * H^2) / (8 * nu * L)
   ```
   where H is the gap height, L is the domain length, and nu is the kinematic viscosity derived from tau.

3. **Symmetric profile.** The velocity profile should be symmetric about the centerline.

4. **No substrate fields.** The output should contain only velocity and pressure fields -- no concentration data.

5. **Convergence.** The Navier-Stokes solver should converge (residual drops below `ns_converge_iT1`) well within `ns_max_iT1` iterations.

## Expected Runtime

Approximately **1 minute** on a standard workstation (single core).

## How to Use

### Using the CompLaB3D GUI

1. **Launch the GUI:**
   ```bash
   cd GUI
   python main.py
   ```

2. **Create a new project:**
   - Click **File > New Project**.
   - Select the **"Flow Only"** template from the template list.
   - Choose a project name and output directory.

3. **Generate geometry:**
   - Go to the **Domain** panel.
   - Set dimensions to **nx=30, ny=15, nz=15** and **dx=1 um**.
   - Click **Generate Geometry** and select **Parallel Plates** as the medium type.
   - The geometry file will be saved to the `input/` folder inside your project.

4. **Verify settings:**
   - In the **Fluid** panel, confirm `delta_P = 2.0e-3` and `tau = 0.8`.
   - In the **General** panel, confirm `biotic_mode = false` and all kinetics toggles are off.
   - The **Chemistry** panel should show **0 substrates**.

5. **Export and compile:**
   - Click **File > Export Project** to write `CompLaB.xml` and any kinetics header files to the project folder.
   - Open a terminal, navigate to the `src/` directory, and run:
     ```bash
     make clean && make
     ```

6. **Run the simulation:**
   - Return to the GUI and go to the **Run** panel.
   - Click **Run Simulation**, or run from the terminal:
     ```bash
     ./complab3d ../path/to/your/CompLaB.xml
     ```

7. **Inspect results:**
   - Open the output VTK files in **ParaView**.
   - Plot the x-velocity component along a vertical line at the channel midpoint.
   - Confirm the parabolic Poiseuille profile.
