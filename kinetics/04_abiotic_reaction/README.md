# Scenario 04: Abiotic First-Order Decay (A -> Product)

## Overview

This scenario couples **flow and transport** with a **first-order abiotic decay reaction** in an open pore domain. A reactant (A) is continuously supplied at the inlet and decays to a product (P) as it is advected and diffused through the domain. It validates that the abiotic kinetics engine correctly computes reaction rates, that product forms in stoichiometric proportion to reactant consumed, and that mass is conserved.

## Reactions

```
A  -->  Product

Rate law:  dA/dt = -k * [A]
           dP/dt = +k * [A]

k = 1.0e-5 [1/s]   (first-order decay rate constant)
```

## Validation Criteria

The reviewer should check the following:

1. **Exponential decay along the flow direction.** At steady state, the reactant concentration should decrease approximately exponentially from left to right. For a plug-flow approximation:
   ```
   [A](x) ~ [A]_inlet * exp(-k * x / u)
   ```
   where u is the mean flow velocity.

2. **Product formation.** The product concentration should increase along the flow direction as the reactant decays. Near the outlet, [P] should be highest.

3. **Mass balance at any cross-section.** At steady state, for any x-position:
   ```
   [A](x) + [P](x) = constant
   ```
   This constant should equal the inlet reactant concentration (1.0 mol/L) because each mole of A that decays produces exactly one mole of P.

4. **No negative concentrations.** Both [A] and [P] must remain non-negative at all times and positions.

5. **Reactant never exceeds inlet concentration.** [A] should never exceed 1.0 mol/L anywhere in the domain.

6. **Numerical stability.** No NaN or Inf values in the output. The reaction rate should be smoothly varying.

Approximately **2 minutes** on a standard workstation (single core).

### About the kinetics files

- **`defineAbioticKinetics.hh`** contains the `defineAbioticRxnKinetics()` function that implements `dA/dt = -k*[A]` and `dP/dt = +k*[A]` with rate clamping for numerical stability. The rate constant `k_decay_0 = 1.0e-5` is set in the `AbioticParams` namespace.

- **`defineKinetics.hh`** is a no-op stub. It defines the required `defineRxnKinetics()` function and the `KineticsStats` namespace (both required by the C++ solver for compilation) but sets all rates to zero.

**Important:** The C++ solver unconditionally `#include`s both header files. Both files must always be present even if one is a no-op.

## How to Use

### Using the CompLaB3D GUI

1. **Launch the GUI:**
   ```bash
   cd GUI
   python main.py
   ```

2. **Create a new project:**
   - Click **File > New Project**.
   - Select the **"Abiotic - First Order Decay"** template from the template list.
   - Choose a project name and output directory.

3. **Generate geometry:**
   - Go to the **Domain** panel.
   - Set dimensions to **nx=30, ny=10, nz=10** and **dx=1 um**.
   - Click **Generate Geometry** and select **Rectangular Channel** (option 1) to create a fully open pore domain.

4. **Verify settings:**
   - In the **General** panel, confirm `biotic_mode = false`, `enable_abiotic_kinetics = true`.
   - In the **Fluid** panel, confirm `delta_P = 2.0e-3` and `tau = 0.8`.
   - In the **Chemistry** panel, confirm 2 substrates:
     - Substrate 0: "Reactant", C0 = 1.0, left Dirichlet = 1.0
     - Substrate 1: "Product", C0 = 0.0, left Dirichlet = 0.0
   - Optionally, click the **Kinetics Editor** button to inspect or modify the abiotic rate law in `defineAbioticKinetics.hh`.

5. **Export and compile:**
   - Click **File > Export Project** to write `CompLaB.xml` and both kinetics header files to the project folder.
   - Open a terminal, navigate to the `src/` directory, and run:
     ```bash
     make clean && make
     ```
   This recompilation links the exported `defineAbioticKinetics.hh` with the solver.

6. **Run the simulation:**
   - Return to the GUI and go to the **Run** panel.
   - Click **Run Simulation**, or run from the terminal:
     ```bash
     ./complab3d ../path/to/your/CompLaB.xml
     ```

7. **Inspect results:**
   - Open the output VTK files in **ParaView**.
   - Plot [Reactant] along the x-axis and verify the exponential decay profile.
   - Plot [Product] along the x-axis and verify the complementary increase.
   - At any x cross-section, compute [A] + [P] and confirm it equals approximately 1.0 mol/L.
